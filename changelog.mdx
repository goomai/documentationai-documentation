<Update label="2026-07-10" description="Real-time collaboration, comments, branch workflows, and SEO fields" tags={["new", "collaboration", "editor"]}>

### New features

- **Real-time collaborative editing** — Multiple teammates can now edit the same page simultaneously with live cursors and presence indicators.
- **Inline comments** — Select any text to start a comment thread. Reply, resolve, and reopen discussions anchored to specific passages.
- **@Mentions & notifications** — Tag teammates with `@` in comments. A bell icon shows unread notifications; mentions and replies trigger real-time toast alerts.
- **Collaborator avatars** — See who's currently editing with live avatar stacks in the editor toolbar.
- **Branch workflows in the editor** — Save to a branch, save & merge, or create a pull request — all without leaving the editor.
- **Preview deployment URLs** — Preview changes on a feature branch before merging (Standard plan and above).
- **Branch-based editor URLs** — Editor links now include the branch name, making them bookmarkable and shareable.

### Updates

- **SEO meta fields** — Documents now support custom meta titles and descriptions that override browser tab titles and search result previews without changing visible headings.
- **Read-only mode for viewers** — Users with a viewer role can observe live edits and comments but cannot make changes.
- **Real-time navigation sync** — Changes to navigation, settings, and redirects propagate to all collaborators instantly.
- **Guided editor upgrade** — Users with unsaved changes in the legacy editor see a one-time prompt to upgrade once their changes are published.
- **Simplified workflow actions** — Workflow run rows now feature a single "View changes" button that opens the diff in the editor.
- **Brand refresh** — The accent color updated from orange to coral across the dashboard.

### Bug fixes

- Branch switching no longer drops the collaboration session.
- Self-publish notifications no longer appear for the user who triggered the publish.
- Moving files to the repository root no longer produces invalid paths.
- Comment popover positioning improved for multi-line selections.
- Deleting individual comment messages now works correctly.


</Update>
