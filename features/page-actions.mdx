## Page Actions

Page Actions controls which quick action buttons appear in the sidebar of your published documentation. Use it to show or hide individual actions based on how you want readers to interact with each page. Changes are saved to your documentation configuration and apply after you publish.

## Available actions

Choose from these sidebar actions, in this order:

- Copy page
- Download as PDF
- View as Markdown
- Open in Claude
- Open in ChatGPT
- Open in Perplexity
- Open in Grok
- Copy MCP server URL

## How it works

<Callout kind="info">

All page actions are enabled by default. If you leave every action on, no extra configuration is needed.

</Callout>

When you turn one or more actions off, the configuration stores only the action IDs that remain enabled. This keeps your documentation configuration clean while still letting you control exactly what appears in the published sidebar.

## Editing page actions

<Steps>
  <Step title="Open Page Actions in the editor" icon="settings" title-type="p">

  Open the editor's settings sidebar and find the **Page Actions** section.

  </Step>
  <Step title="Toggle each action on or off" icon="toggle-left" title-type="p">

  Turn individual switches on or off for each action you want to show or hide in the published sidebar.

  </Step>
  <Step title="Publish your documentation" icon="upload" title-type="p">

  Publish your documentation to apply the updated sidebar actions to the live site.

  </Step>
</Steps>

<Callout kind="alert">

Changes do not affect the live sidebar until you publish.

</Callout>
